Environment and configuration management.

"Environment" is just another word for configuration, but the idea is that everything that could affect the way the bot makes configuration-based decisions should be encapsulated in the structures found in this class, most notably Env.

Since the environment must be loaded before anything else can happen, the load takes no arguments. Anything that could affect the way the environment is loaded should itself be part of the environment, so the logic for detecting those things should live in this module.
